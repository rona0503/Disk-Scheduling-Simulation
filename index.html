▸ Knowledge Base
① SEEK TIME
Time for the r/w head to move to the correct track/cylinder. Dominant cost. Range: 1–20ms. This is what scheduling minimizes.
② ROTATIONAL LATENCY
Time for the correct sector to spin under the head. Avg = ½ rotation ≈ 2–8ms. Not influenced by scheduling.
③ DATA TRANSFER
Time to read/write once positioned. Proportional to data size. Usually smallest component.
TOTAL ACCESS TIME =
Seek + Rotational Latency + Transfer
④ REAL-WORLD ANALOGY
Disk scheduling is like an elevator deciding which floors to stop at. FCFS = stop in the order buttons were pressed. SSTF = stop at closest floor first. SCAN = sweep up, then down — just like a real elevator.
⑤ OS USAGE
Linux uses CFQ / BFQ (fair queuing). Windows uses a variant of LOOK. SSDs make seek time negligible — scheduling matters less for modern NVMe drives.
FCFS
Use when fairness is critical and throughput is secondary. Good for small queues or debugging.
Avoid: High-load systems needing performance.
SSTF
Use for maximum throughput in moderate-load systems where some request delay is acceptable.
Avoid: Any system where fairness guarantees are needed (edge requests starve).
SCAN / LOOK
Best general-purpose choice. Balanced performance + fairness. LOOK is more efficient (doesn't overshoot disk boundaries).
Use for: General-purpose OS disk scheduling.
C-SCAN / C-LOOK
Best for high-load servers. Uniform wait time across all cylinders. C-LOOK optimizes by not sweeping to disk boundaries.
Use for: Database servers, high-throughput I/O systems.
